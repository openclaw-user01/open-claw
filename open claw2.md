多数部署失败的原因，都是安全防护软件拦截了程序核心文件。正式安装开始前，请彻底退出 360 安全卫士、电脑管家、火绒等防护工具，同时临时关闭 Windows Defender 实时防护。

该软件需要调用键鼠模拟、文件读写等系统底层权限，容易被安全工具误判为风险程序并隔离组件。该项目为开源项目，仅在安装阶段临时关闭防护，不会产生安全问题。

## ⚙️ 五、自动化安装配置分步讲解
OpenClaw v2.7.9 对部署逻辑做了全面优化，全程自动化配置，不需要手动修改任何配置文件。

1. 在软件欢迎界面点击**开始使用**，跳转至安装路径设置页面；
2. 路径强制规范：目录必须为纯英文，禁止出现空格、中文、特殊符号。

✅ 推荐路径示例：
D:\AItools\OpenClaw
E:\OpenClaw_v2.7.9
plaintext

❌ 不建议安装在 C 盘，会占用系统盘空间，影响电脑运行速度。

3. 勾选用户协议与免责声明，点击**开始安装**，整体流程耗时 3~5 分钟；
4. 程序会自动补全运行组件、适配系统设置、生成配置文件并创建桌面快捷方式，安装期间请勿关闭窗口，否则会直接中断部署。

### 第一次启动初始化说明
安装完成后客户端会自动弹出，第一次运行 Gateway 服务需要加载初始化资源，界面会提示等待服务就绪，耐心等候 1~3 分钟即可正常使用，后续启动无需重复等待。

## 🛠️ 七、常见问题快速处理方案
### 启动文件被杀毒软件隔离
退出所有安全防护软件，在隔离区恢复文件，重新解压安装包并按照流程部署。

### 提示路径非法，安装终止
更换纯英文、无特殊字符的目录，重新执行安装操作。

### Gateway 一直处于离线状态，无法发送指令
确认安全防护已全部关闭、安装路径符合规范，点击客户端重启服务；若问题依旧，完全关闭程序后重新运行启动文件。

### 第一次启动加载缓慢
属于正常的初始化过程，等待片刻即可，再次启动时速度会明显提升。
